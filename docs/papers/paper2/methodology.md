## 6.1 Pre-registered hypotheses (OSF, inherited from Paper 1)

The four pre-registered hypotheses are **inherited from
Paper 1** — cycle 2 does *not* register new hypotheses, because
Paper 2 is a replication artifact, not a fresh empirical claim.
The OSF lock (DOI pending per Paper 1 §6.1) applies verbatim.

- **H1 — Forgetting reduction.**
  `mean(forgetting_P_equ) < mean(forgetting_baseline)`.
  Test : Welch's one-sided t-test.
- **H2 — P_max equivalence.**
  `|mean(acc_P_max) - mean(acc_P_equ)| < 0.05`.
  Test : two one-sided tests (TOST), ε = 0.05.
- **H3 — Monotonic alignment.**
  `mean(acc_P_min) < mean(acc_P_equ) < mean(acc_P_max)`.
  Test : Jonckheere-Terpstra.
- **H4 — Energy budget.**
  `mean(energy_dream / energy_awake) < 2.0`.
  Test : one-sample t-test.

Cycle-2 scope note : with P_max fully wired (G8,
`docs/milestones/g8-p-max-functional.md`), H2 and H3 now
admit a genuine three-group structure — but the numerical
evaluation is still synthetic-substitute because the
predictor is shared across substrate labels (§6.4).

## 6.2 Statistical pipeline + Bonferroni α

All four tests run under a **Bonferroni-corrected significance
threshold** : `α_per_hypothesis = 0.05 / 4 = 0.0125`. The
implementation is the cycle-1 `kiki_oniric.eval.statistics`
module (unchanged in cycle 2 — intentionally, so the cycle-2
p-values are numerically comparable to the cycle-1 values
when the same predictor is used) :

- **`welch_one_sided`** (H1) : `scipy.stats.ttest_ind` with
  `equal_var=False`, p-value halved for one-sided
  interpretation.
- **`tost_equivalence`** (H2) : two manual one-sided t-tests
  (lower bound `diff ≤ −ε`, upper bound `diff ≥ +ε`). Reject
  H0 when both pass at α (TOST max-p rule).
- **`jonckheere_trend`** (H3) : sum of pairwise Mann-Whitney U
  counts across ordered groups, z-approximation for p-value.
- **`one_sample_threshold`** (H4) : `scipy.stats.ttest_1samp`
  against `popmean = threshold`, one-sided adjustment for
  sample below threshold.

All four return a uniform `StatTestResult(test_name, p_value,
reject_h0, statistic)` for downstream handling.

## 6.3 Ablation matrix : 2 substrates × 3 profiles × 3 seeds = 18 cells

Paper 2's matrix dimension is new : Paper 1 ran 3 profiles ×
3 seeds = 9 cells. Paper 2 crosses substrates, giving
**2 × 3 × 3 = 18 cells**, plus a **fourth standalone row** for
the Wake-Sleep CL baseline [@alfarano2024wakesleep, IEEE TNNLS,
arXiv 2401.08623] scored on the same 3 seeds (`[42, 123, 7]`)
— see §5.8 + dump
`docs/milestones/wake-sleep-baseline-2026-05-03.json` (variant
c, published reference values, Paper 2 §6.4 caveat applies).
The baseline scores only on M1.* metrics (`forgetting_rate`,
`avg_accuracy`) ; it does not enter H2/H3/H4 because it does
not produce a P_min/P_equ/P_max grouping. Each substrate-cell
still produces :

- accuracy (for H1, H2, H3)
- forgetting := 1 − accuracy (for H1)
- energy ratio := energy_dream / energy_awake (for H4)

The cells are generated by `scripts/ablation_cycle2.py` ;
Markdown + JSON dumps land at
`docs/milestones/cross-substrate-results.{md,json}`.

Seed grid : `[42, 123, 7]` — identical to cycle 1, so the
cycle-2 MLX row is bit-comparable to Paper 1 §7.2 when the
predictor is unchanged.

## 6.4 Synthetic-substitute predictor (the critical caveat)

**(synthetic substitute — not empirical claim.)**  This
subsection is load-bearing for reading §7 correctly.

The two substrate rows in the 2 × 3 × 3 matrix share the same
Python mock predictor. The MLX substrate row and the E-SNN
substrate row pass their state representations (MLX arrays
vs LIFState) through the 4 op factories and emit the expected
output channels — but when `evaluate_retained` queries for a
per-item prediction, the same mock function returns the same
label regardless of which substrate produced the state.

Three consequences follow :

1. The **per-substrate p-values are bit-identical** in the
   limit of a perfect shared predictor, and very close in
   practice (see §7 table). This agreement is **not** evidence
   of substrate-agnostic performance on real data ; it is the
   *absence* of a substrate-specific signal in the predictor.
2. The cross-substrate **agreement verdict** (4 / 4 agree) is
   trivially YES by construction.
3. The **architectural claim** — that the pipeline
   (runner → stats → dump) executes end-to-end on two
   distinct substrate registrations without code duplication
   — is validated. This is the half of DR-3 Conformance that
   Paper 2 earns empirically.

A divergent-predictor replication (substrate-specific
inference : MLX forward pass on MLX state, LIFState read-out
on LIFState) is the **cycle-3 target**. Paper 2 does not make
that claim.

## 6.5 Reproducibility : R1 contract + benchmark integrity

The cycle-2 matrix is reproducible by the same two contracts
as cycle 1 :

- **R1 (deterministic run_id)** — every cell resolves to a
  32-hex SHA-256 prefix of `(c_version, profile, seed,
  commit_sha)`. The cycle-2 batch wrapper and the ablation
  runner both emit run_ids recorded in the JSON dump :
  - `cycle2_batch_id` : `3a94254190224ca82c70586e1f00d845`
  - `ablation_runner_run_id` :
    `45eccc12953e758440fca182244ddba2`
  - `harness_version` : `C-v0.6.0+PARTIAL`
- **R3 (benchmark integrity)** — the mega-v2 stratified
  500-item retained benchmark ships with a `.sha256` file ;
  the loader rejects any item file whose hash does not match
  the frozen reference (raising `RetainedIntegrityError`).
  Cycle-2 uses the synthetic fallback (flagged
  `synthetic:c8a0712000b641...`) inherited from cycle 1,
  pending real mega-v2 ablation in a later cycle.

The DualVer tag `C-v0.6.0+PARTIAL` is attached to every
cycle-2 artifact. Empirical claims are **valid only against
their declared `c_version`** ; bumping the formal axis MAJOR
would invalidate empirical-axis artifacts and require a
re-run.

## 6.6 What changed vs Paper 1 methodology

Paper 2 is deliberately minimal on methodology additions :

- **Same hypotheses** (H1-H4, OSF-locked).
- **Same statistical module** (no new tests, no new
  corrections).
- **Same seed grid** (`[42, 123, 7]`).
- **Same Bonferroni α** (0.0125).
- **Same predictor class** (shared mock, synthetic substitute).

What is **new** in Paper 2 :

- Substrate dimension in the matrix (2 rows).
- Cross-substrate agreement verdict (aggregate across H1-H4).
- Conformance matrix artifact wired to the runner
  (`scripts/conformance_matrix.py`).

This discipline — change one dimension at a time — is how
a replication paper preserves the interpretability of its
comparisons. Cycle-3 will add the predictor dimension ;
cycle-2 restricts itself to the substrate dimension under a
fixed predictor.
